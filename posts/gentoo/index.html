May 24, 2023
7 min read

Nesecitas esta Herramienta

una live cd con linux : arch,mint,gentoo mininal o lo que sea.

Guia instaclion Gentoo Linux:

sudo cfdisk /dev/sda (crear particiones)

setear formato a partciones: sudo mkfs.vfat -F 32 /dev/sda1

sudo mkfs.vfat -F 32 /dev/sda1 sudo mkfs.ext4 /dev/sda3

no usar este comando a menos que se use en partciones menores de 8gb de disco:
mkfs.ext4 -T small /dev/device

crear formato swap a la particion: sudo mkswap /dev/sda2

activar la swap de memoria en la particion: sudo swapon /dev/sda2

crear punto de montaje en gentoo: sudo mkdir –parents /mnt/gentoo

montar particion raiz en gentoo sudo mount /dev/sda3 /mnt/gentoo

entrar en el directorio del sistema: cd /mnt/gentoo

bajar tallbar por links o de forma natural desde mirrors distfiles: sudo links https://www.gentoo.org/downloads/mirrors/ o con wget con el nombre del archivo

descomprimir tallbar en la raiz principal: sudo tar xpvf stage3-.tar.xz –xattrs-include=’.*’ –numeric-owner

editar portage:

nano -w /mnt/gentoo/etc/portage/make.conf

sudo cp /mnt/gentoo/usr/share/portage/config/repos.conf /mnt/gentoo/etc/portage/repos.conf/gentoo.conf

sudo mount --bind /run /mnt/gentoo/run
sudo mount --make-slave /mnt/gentoo/run

instalar bootloader mbr:

    sudo emerge --ask --verbose sys-boot/grub
     
       
        sudo emerge --ask --verbose sys-boot/grub2 (opcional)

A note for UEFI users: running the above command will output the enabled GRUB_PLATFORMS values before emerging. When using UEFI capable systems, users will need to ensure GRUB_PLATFORMS="efi-64" is enabled (as it is the case by default). If that is not the case for the setup, GRUB_PLATFORMS="efi-64" will need to be added to the /etc/portage/make.conf file before emerging GRUB so that the package will be built with EFI functionality:

    sudo echo 'GRUB_PLATFORMS="efi-64"' >> /etc/portage/make.conf

    sudo emerge --ask sys-boot/grub
    
    sudo emerge --ask sys-boot/grub2   (opcional)

    If GRUB was somehow emerged without enabling GRUB_PLATFORMS="efi-64", the line (as shown above) can be added to make.conf and then dependencies for the world package set can be re-calculated by passing the --update --newuse options to emerge:

    sudo emerge --ask --update --newuse --verbose sys-boot/grub

The GRUB software has now been merged to the system, but not yet installed.
Install

Next, install the necessary GRUB files to the /boot/grub/ directory via the grub-install command. Presuming the first disk (the one where the system boots from) is /dev/sda, one of the following commands will do:

   importante cuando usas bios clasica mbr:

    sudo grub-install /dev/sda


    importante cuando usas efi:

    sudo grub-install --target=x86_64-efi --efi-directory=/boot
    
    
    generar config grub:
    sudo grub-mkconfig -o /boot/grub/grub.cfg
    sudo grub2-mkconfig -o /boot/grub/grub.cfg (opcional)
    
    reniciar y desmontar el sistema:
    Rebooting the system
